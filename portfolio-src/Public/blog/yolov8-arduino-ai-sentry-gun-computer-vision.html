Building a YOLOv8 tracking system with Arduino — and what it took to make it physical

An AI sentry gun that detects objects, recognises faces, and physically follows its target using pan–tilt servos and a laser module. Here's the full build walkthrough — hardware decisions, software integration, and every mistake in between.

This is a project about computer vision hardware integration done for real — not a screen demo, but a physical system that moves. The core idea: combine a YOLOv8 real-time object tracking pipeline with an Arduino-controlled servo rig, add OpenCV face recognition, wire in a laser diode, and build a sentry turret that reacts to the world. Two weeks, one pan–tilt mount, four target keywords, and more jitter than I expected.

Most CV tutorials stop at the bounding box. This one starts there and goes all the way to hardware firing a laser on a locked face. Here's everything, in the order it actually happened.

What this AI sentry gun project actually does

Three distinct operating modes, each doing something meaningfully different:

👤
Show Enemy
Capture face images and train a custom recogniser. Enter a name, look at the camera, system learns that person.
🎯
Object Shooting
YOLOv8 tracks any object class — person, cup, phone. Turret physically follows. Laser activates on lock.
🔍
Track Enemy
Face recognition mode. Hunts for trained faces specifically, ignoring everyone else in the frame.

YOLOv8 gives class-level detection. Face recognition gives identity-level targeting. They're different problems, different pipelines, coexisting in one system.

YOLOv8 tracking system: how the detection logic works

YOLOv8 returns bounding boxes. Extract the center, compare to frame center — the difference is the error. The servos exist to drive that error to zero.

Core error calculation
error_x = target_center_x - frame_center_x error_y = target_center_y - frame_center_y # System goal: drive both toward zero, every frame
Arduino object tracking via PyFirmata2 — pin mapping
from pyfirmata2 import Arduino board = Arduino('COM7') pan_pin = board.get_pin('d:9:s') # X-axis servo tilt_pin = board.get_pin('d:10:s') # Y-axis servo laser = board.get_pin('d:11:o') # laser / relay module if abs(error_x) > dead_zone: pan_angle = max(0, min(180, pan_angle + step_x)) pan_pin.write(pan_angle) laser.write(1 if locked else 0) # engage on lock

Don't write to the servos every frame — this floods the Arduino and causes constant jitter. Update only when the error exceeds a threshold, with a short delay between commands. The smoothness difference is dramatic.

OpenCV face recognition: identity-level targeting

The LBPH recogniser is trained in-app — no cloud, no API, entirely offline. You capture face images, assign a name, and the model distinguishes that person from anyone else. Fast, private, but sensitive to lighting: capture training images in the same environment you'll actually operate in.

LBPH accuracy drops significantly with lighting changes between training and deployment. Consistent, even light is the single biggest factor — more than distance, more than angle.

Laser and relay engagement — the hardware output

A laser diode wired to Arduino pin 11 through a relay closes when the target is locked within the dead zone. Manual keypress override also works. Requiring N consecutive locked frames before engaging prevents false triggers entirely.

Even low-power laser diodes cause eye damage at close range. Use a paper target on a wall, never a person's face. Ensure StandardFirmataPlus is loaded onto the Arduino before connecting PyFirmata2.

Servo selection: the part nobody tells you

SG90 micro servos are in every tutorial. They're also wrong for this build the moment you add any real load.

Avoid
SG90 micro servo
  • Low torque
  • Can't handle load
  • High jitter under stress
  • Platform shakes constantly
Use this instead
MG996R servo
  • Higher torque
  • Handles load well
  • Smooth controlled motion
  • Minimum viable for this build

Computer vision hardware integration: four real-time problems

Latency
Keep the detection loop lean. Profile face recognition overhead separately from the servo update loop.
Jitter
Apply a moving average to bounding box centers. Motors shouldn't react to pixel-level frame-to-frame noise.
Overcorrection
Dead zone + step limit. Ignore small errors; cap how far the servo moves per frame. Stops oscillation.
False triggers
Confidence threshold + N consecutive locked frames before laser engages. Fixed premature firing completely.
"Machine learning coursework trains you to optimise a model. Nobody teaches you to think about the system around it — the hardware, the latency, the control loop. That's where the real problems live."

What to build next

PID control to replace threshold-based servo logic. A faster inference pipeline — model quantisation or a Jetson Nano. FaceNet instead of LBPH for lighting-robust face recognition. And if you're reading this as someone who wants to build the same thing: all code is on GitHub.

Starting fresh: use MG996R servos, wire the laser through a relay, upload StandardFirmataPlus before touching PyFirmata2, and capture face training images in your actual operating environment. In that order.
